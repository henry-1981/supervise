# Output Styles Documentation

## Overview

Cowork Supervisor v1.1.0 introduces Output Styles - a flexible system for customizing how responses are formatted and delivered. This enables the supervisor to adapt to different contexts, from professional project management to technical code reviews.

## Format Conversion

### Supported Formats

| Format | Extension | Features |
|--------|-----------|----------|
| **Markdown** | `.md` | Syntax highlighting, full formatting |
| **HTML** | `.html` | Web-ready, semantic markup, accessibility |
| **PDF** | `.pdf` | Print-ready, bookmarks, table of contents |
| **Word** | `.docx` | Track changes, company templates |

### Conversion Pipeline

```

    â†“ Pandoc / wkhtmltopdf
HTML (Web Preview)
    â†“ PDF Generation

    â†“ docx conversion
Word (Internal Review)
```

### Conversion Script

```bash
# Convert to HTML with accessibility
.moai/scripts/convert-output.sh \
  --format html \
  --input report.md \
  --output report.html \
  --style moai

# Convert to PDF with company branding
.moai/scripts/convert-output.sh \
  --format pdf \
  --input report.md \
  --output report.pdf \
  --template company-template

# Validate accessibility
.moai/scripts/validate-accessibility.sh report.html
```

## Configuration

### Output Configuration File

Create `.moai/config/sections/output.yaml`:

```yaml
# Output Configuration
output:
  # Default format for conversions
  default_format: markdown

  # Default style for responses
  default_style: moai

  # Format-specific settings
  formats:
    markdown:
      extension: ".md"
      encoding: "utf-8"
      line_width: 80

    html:
      extension: ".html"
      css_theme: "regulatory-standard"
      accessibility: "wcag-2.1-aa"
      include_toc: true
      syntax_highlighting: true

    pdf:
      extension: ".pdf"
      page_size: "letter"
      margins: "0.75in"
      toc: true
      bookmarks: true
      metadata:
        author: "Company Name"
        subject: "Regulatory Submission"

    word:
      extension: ".docx"
      template: "regulatory-template.dotx"
      track_changes: false

  # Style-specific settings
  styles:
    moai:
      language_aware: true
      show_progress: true
      bilingual_support: true
      emoji_decorations: true

    r2d2:
      concise: true
      technical: true
      code_focused: true
      minimal_formatting: true

    yoda:
      wisdom_focused: true
      strategic: true
      thoughtful: true
      educational: true
```

## Accessibility Compliance

### WCAG 2.1 AA Standards

HTML output generated by Cowork Supervisor complies with:

- **Perceivable**: Semantic markup, alt text, high contrast
- **Operable**: Keyboard navigation, skip links
- **Understandable**: Clear language, consistent navigation
- **Robust**: ARIA labels, proper HTML structure

### Validation

## Usage Examples

### Selecting Output Style

```bash
# Use MoAI style for professional coordination
/supervise "Prepare quarterly business review" --style moai

# Use R2D2 style for technical analysis
/supervise "Review code for security vulnerabilities" --style r2d2

# Use Yoda style for strategic planning
/supervise "Analyze market entry strategy" --style yoda
```

### Converting Output Format

```bash
# Convert to HTML
/supervise "Generate market analysis" --format html --output market.html

# Convert to PDF
/supervise "Create regulatory submission" --format pdf --output submission.pdf

# Convert to Word
/supervise "Draft internal report" --format word --output report.docx
```

### Combined Options

```bash
# Generate with style and format
/supervise "Prepare executive summary" \
  --style moai \
  --format pdf \
  --output summary.pdf \
  --template executive-template
```

## Integration with Plugins

Output styles are automatically applied to results from all coordinated plugins:

```bash
# All plugin results formatted in R2D2 style
/supervise "Analyze technical stack" --style r2d2

# Results from Finance, Legal, and Marketing plugins
# all formatted consistently in selected style
```

## Custom Templates

### Company Branding

Create custom templates for PDF/Word output:

```bash
# Create company template directory
mkdir -p .moai/templates/company

# Add template files
cp company-logo.png .moai/templates/company/
cp company-font.ttf .moai/templates/company/
```

### Template Configuration

```yaml
# In output.yaml
pdf:
  template: "company-template"
  branding:
    logo: ".moai/templates/company/company-logo.png"
    colors:
      primary: "#0066CC"
      secondary: "#333333"
    fonts:
      body: "company-font.ttf"
      headings: "company-font-bold.ttf"
```

## Performance Considerations

### Conversion Speed

| Format | Speed | Notes |
|--------|-------|-------|
| Markdown | Instant | No conversion needed |
| HTML | Fast | <1 second for typical documents |
| PDF | Medium | 1-5 seconds depending on complexity |
| Word | Medium | 2-10 seconds with templates |

### Optimization Tips

1. **Use Markdown for drafts**: Fast iteration, instant preview
2. **Convert to HTML for web**: Fast loading, accessible
3. **Generate PDF for final**: Print-ready, professional
4. **Use Word for reviews**: Track changes, collaboration

## Troubleshooting

### Common Issues

**Issue**: HTML conversion fails
- **Solution**: Check Pandoc installation: `pandoc --version`

**Issue**: PDF has no bookmarks
- **Solution**: Ensure `toc: true` in output.yaml

**Issue**: Accessibility validation fails
- **Solution**: Check for missing alt text, low contrast

**Issue**: Word template not applied
- **Solution**: Verify template path in output.yaml

## 1. Purpose

The purpose of this procedure is to:

- Define the process for identifying, documenting, investigating, and correcting nonconformities
- Define the process for identifying and implementing preventive actions
- Ensure CAPA activities are effective and do not adversely affect product safety or performance
- Ensure CAPA activities are documented and verified
- Prevent recurrence of nonconformities
- Prevent occurrence of potential nonconformities

---

## 2. Scope

This procedure applies to:

- All nonconformities identified within the QMS
- Customer complaints requiring investigation
- Nonconforming product
- Internal audit findings
- External audit findings
- Management review action items
- Process deviations
- Trends indicating potential quality issues
- Suppliers and subcontractors

| **Management Representative** | Review and approval of significant CAPA plans |
| **Affected Personnel** | Participation in CAPA investigations and implementation |
| **Document Control** | Maintenance of CAPA records and documents |

---

## 4. Definitions

| Term | Definition |
|------|------------|
| **Correction** | Action to eliminate a detected nonconformity |
| **Corrective Action** | Action to eliminate the cause of a detected nonconformity to prevent recurrence |
| **Preventive Action** | Action to eliminate the cause of a potential nonconformity to prevent occurrence |
| **Nonconformity** | Non-fulfillment of a requirement |
| **Root Cause** | The fundamental reason for a nonconformity |
| **Verification** | Confirmation that specified requirements have been met |
| **Validation** | Confirmation that requirements for an intended use have been met |
| **Effectiveness Check** | Evaluation to confirm that CAPA actions were effective |

### 6.2 CAPA Initiation

#### 6.2.1 Problem Identification

**Sources of CAPA Initiations**:

| Source | Trigger | Documentation |
|--------|---------|---------------|
| **Customer Complaints** | Complaint received | Customer complaint record |
| **Nonconforming Product** | Nonconformity detected | Nonconformance report |
| **Internal Audit** | Audit finding | Internal audit report |
| **External Audit** | Audit finding | External audit report |
| **Management Review** | Action item | Management review minutes |
| **Process Deviation** | Deviation observed | Deviation report |
| **Data Trend** | Trend analysis report | Trend analysis report |
| **Supplier Issue** | Supplier nonconformity | Supplier nonconformance report |
| **Field Safety** | FSCA or advisory notice | Field safety notice |

**CAPA Initiation Criteria**:

A CAPA is initiated when:
- A nonconformity is detected that requires investigation
- A trend indicates a potential quality issue
- Customer complaints indicate systemic issues
- Audit findings require corrective action
- Management identifies improvement opportunities

#### 6.2.2 CAPA Request

**CAPA Request Form**: Form FM-8.5.X-1 CAPA Request

**CAPA Request Contents**:

1. **Problem Description**:
   - Clear description of the nonconformity or issue
   - Date of occurrence
   - Location of occurrence
   - Person who identified the issue

2. **Immediate Actions Taken**:
   - Containment actions implemented
   - Product affected
   - Customers affected

3. **Severity Assessment**:
   - Impact on product safety
   - Impact on product performance
   - Regulatory impact
   - Customer impact

4. **Prioritization**:
   - Priority level (Critical, High, Medium, Low)
   - Target completion date

**CAPA Request Approval**:

| Severity | Approval Required | Timeline for Review |
|----------|-------------------|---------------------|
| Critical | Quality Director, Management Representative | Within 24 hours |
| High | Quality Director | Within 3 working days |
| Medium | CAPA Coordinator | Within 5 working days |
| Low | CAPA Coordinator | Within 10 working days |

### 6.3 Investigation and Root Cause Analysis

#### 6.3.1 Investigation Planning

**Investigation Team**: Assembled based on issue complexity and severity

**Investigation Plan**:

1. **Investigation Scope**:
   - Processes affected
   - Products affected
   - Time period to investigate

2. **Data Collection Plan**:
   - Records to review
   - Personnel to interview
   - Samples to collect
   - Tests to conduct

3. **Investigation Timeline**:
   - Data collection deadline
   - Analysis deadline
   - Report deadline

#### 6.3.2 Data Collection

**Data Collection Methods**:

| Method | When Used | Examples |
|--------|-----------|----------|
| **Record Review** | Nonconformity documented | Batch records, test reports, maintenance logs |
| **Interviews** | Human factors involved | Operators, supervisors, customers |
| **Observation** | Process-related issues | Direct observation of process |
| **Testing** | Product-related issues | Additional testing of samples |
| **Statistical Analysis** | Trends or patterns | SPC data, trend analysis |

**Data Collection Checklist**:

- [ ] All relevant records collected
- [ ] Interviews conducted
- [ ] Observations documented
- [ ] Testing completed (if applicable)
- [ ] Statistical analysis performed (if applicable)

#### 6.3.3 Root Cause Analysis Methods

**Method Selection**:

Based on the nature of the nonconformity, select appropriate root cause analysis method(s):

| Method | Best For | Description |
|--------|----------|-------------|
| **5 Whys** | Simple, single-cause issues | Ask "why" five times to reach root cause |
| **Fishbone (Ishikawa)** | Complex, multi-factor issues | Categorize potential causes |
| **Fault Tree Analysis** | Safety-critical issues | Top-down deductive analysis |
| **Cause and Effect Matrix** | Process-related issues | Prioritize potential causes |
| **Change Analysis** | Issues following changes | Compare before and after change |
| **Brainstorming** | Complex issues with multiple possible causes | Group ideation |

### 6.4 CAPA Planning

#### 6.4.1 CAPA Action Plan

**Form**: FM-8.5.X-4 CAPA Action Plan

**CAPA Action Plan Contents**:

1. **CAPA Objectives**: Clear, measurable objectives
2. **CAPA Actions**: Specific actions to address root cause(s)
3. **Responsibility**: Person responsible for each action
4. **Timeline**: Target completion date for each action
5. **Resources**: Resources required (personnel, equipment, budget)
6. **Verification Method**: How implementation will be verified
7. **Effectiveness Check Method**: How effectiveness will be evaluated

#### 6.4.2 CAPA Action Types

**Correction vs. Corrective Action**:

| Action Type | Purpose | Examples |
|-------------|---------|----------|
| **Correction** | Fix immediate problem | Rework, repair, replacement |
| **Corrective Action** | Prevent recurrence | Process change, training, procedure update |
| **Preventive Action** | Prevent occurrence | Risk-based preventive measures |

**CAPA Action Categories**:

| Category | Examples |
|----------|----------|
| **Design Changes** | Design revision, design verification |
| **Document Changes** | Procedure updates, work instruction updates |
| **Process Changes** | Process modification, new process steps |
| **Equipment Changes** | Equipment upgrade, additional equipment |
| **Training** | Operator training, retraining |
| **Supplier Changes** | Supplier qualification, supplier change |
| **System Changes** | Quality system improvements |

### 6.5 CAPA Implementation

#### 6.5.1 CAPA Implementation Tracking

**CAPA Log**: FM-8.5.X-5 CAPA Log

**CAPA Log Contents**:

| CAPA Number | Description | Severity | Status | Open Date | Target Close | Actual Close |
|-------------|-------------|----------|--------|-----------|--------------|-------------|
| CAPA-XXXX-001 | [Description] | [Severity] | [Status] | [Date] | [Date] | [Date] |

**CAPA Status Definitions**:

| Status | Definition |
|--------|------------|
| **Open** | CAPA initiated, investigation pending |
| **In Investigation** | Investigation in progress |
| **Planning** | Root cause identified, action plan being developed |
| **In Progress** | Actions being implemented |
| **Verification** | Implementation verified, effectiveness check pending |
| **Closed** | CAPA completed and verified effective |
| **On Hold** | CAPA temporarily suspended |
| **Cancelled** | CAPA cancelled with justification |

#### 6.5.2 Implementation Monitoring

**Monitoring Activities**:

- Regular progress meetings with implementation team
- Progress updates in CAPA log
- Escalation of overdue CAPAs
- Status reporting to management

**Monitoring Frequency**:

| Priority | Progress Updates | Management Reporting |
|----------|------------------|----------------------|
| Critical | Daily | Weekly |
| High | Weekly | Biweekly |
| Medium | Biweekly | Monthly |
| Low | Monthly | Quarterly |

### 6.6 CAPA Verification

#### 6.6.1 Implementation Verification

**Verification Purpose**: Confirm that CAPA actions were implemented as planned

**Verification Methods**:

| Action Type | Verification Method |
|-------------|---------------------|
| **Document Changes** | Review of revised documents |
| **Process Changes** | Observation of new process |
| **Training** | Training records, competency assessment |
| **Equipment Changes** | Installation, qualification |
| **Design Changes** | Design verification records |

**Verification Documentation**:

- Verification report completed
- Evidence of implementation attached
- Verification sign-off by responsible person

#### 6.6.2 Effectiveness Check

**Effectiveness Check Purpose**: Confirm that CAPA actions were effective in preventing recurrence

**Effectiveness Check Timing**:

| Priority | Effectiveness Check Timing |
|----------|----------------------------|
| Critical | Immediately after implementation + 30 days |
| High | 30 days after implementation |
| Medium | 60 days after implementation |
| Low | 90 days after implementation |

**Effectiveness Check Methods**:

| Method | When Used |
|--------|-----------|
| **Trend Analysis** | Recurrent issues, data-driven problems |
| **Monitoring** | Process changes, training |
| **Audit** | Systemic changes, multiple issues |
| **Customer Feedback** | Customer-related issues |
| **Review of Records** | Documentation-related issues |

**Effectiveness Check Criteria**:

- Root cause eliminated or controlled
- No recurrence of nonconformity
- No new issues introduced
- Process stable and controlled

**Effectiveness Check Results**:

| Result | Action |
|--------|--------|
| **Effective** | CAPA closed |
| **Partially Effective** | Additional CAPA actions planned |
| **Not Effective** | Re-initiate investigation and CAPA |

### 6.7 CAPA Closure

#### 6.7.1 Closure Criteria

**CAPA Closure Checklist**:

- [ ] Root cause identified and documented
- [ ] CAPA actions implemented as planned
- [ ] Implementation verified
- [ ] Effectiveness check completed
- [ ] Effectiveness confirmed (CAPA effective) OR
- [ ] Additional actions planned (if partially effective)
- [ ] All documentation completed
- [ ] CAPA log updated
- [ ] Approvals obtained

#### 6.7.2 Closure Approval

**CAPA Closure Approval**:

| Priority | Approval Required |
|----------|-------------------|
| Critical | Quality Director, Management Representative |
| High | Quality Director |
| Medium | CAPA Coordinator |
| Low | CAPA Coordinator |

#### 6.7.3 Closure Documentation

**CAPA Closure Report**:

- CAPA number and title
- Problem description
- Root cause summary
- Actions taken
- Verification results
- Effectiveness check results
- Conclusion
- Lessons learned

**CAPA Closure Sign-off**:

- CAPA Coordinator
- Process Owner
- Quality Director (for high-severity CAPAs)

### 6.8 CAPA Management Review

#### 6.8.1 CAPA Metrics

**CAPA Performance Metrics**:

| Metric | Calculation | Target |
|--------|-------------|--------|
| **CAPA Closure Rate** | Closed CAPAs / Opened CAPAs × 100% | ≥90% annually |
| **On-Time Closure Rate** | On-time CAPAs / Closed CAPAs × 100% | ≥85% |
| **Average Closure Time** | Sum of closure times / Number of closed CAPAs | ≤60 days (high priority) |
| **Recurrence Rate** | Recurred CAPAs / Closed CAPAs × 100% | ≤5% |
| **Effectiveness Rate** | Effective CAPAs / Closed CAPAs × 100% | ≥95% |

#### 6.8.2 CAPA Trend Analysis

**Quarterly CAPA Trend Analysis**:

- CAPAs by department
- CAPAs by category
- CAPAs by severity
- CAPAs by root cause category
- CAPAs by product line
- CAPAs by supplier

**Trend Analysis Output**:

- CAPA trend report
- Identification of systemic issues
- Recommendations for preventive action
- Management review presentation

1st Why: Why did the device fail?
   Answer: Output voltage was below specification.

2nd Why: Why was output voltage below specification?
   Answer: Power supply component was defective.

3rd Why: Why was the power supply component defective?
   Answer: Component was from a lot with known quality issues.

4th Why: Why was a component from a defective lot used?
   Answer: Incoming inspection did not detect the defect.

5th Why: Why did incoming inspection not detect the defect?
   Answer: Inspection procedure did not include this test.

Root Cause: Incoming inspection procedure inadequate

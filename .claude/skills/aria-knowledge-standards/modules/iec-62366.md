# IEC 62366-1:2015 - Usability Engineering

## Overview

IEC 62366-1:2015은 의료기기 유용성 공학 (Usability Engineering)에 대한 국제 표준입니다. 사용자 인터페이스 설계, 사용 오용 분석, 유용성 평가 프로세스를 정의합니다.

## Key Concepts

### Use Error vs User Error

**Use Error (사용 오용)**:
- 사용자 인터페이스 문제로 인한 오류
- 제조사가 통제 가능
- 예: 버튼 배치 혼란, 경고 부족

**User Error (사용자 오류)**:
- 사용자의 의도적 오조작
- 제조사 통제 범위 밖
- 예: 고의적인 규정 위반

### User Interface (UI)

**Definition**: 사용자와 의료기기 간의 모든 상호작용 요소

**UI Components**:
- Physical controls (buttons, switches, knobs)
- Display screens (monitors, touchscreens)
- Labels and markings
- Instructions for use (IFU)
- Packaging
- Software interfaces
- Connectors and accessories

## Usability Engineering Process

### Clause 5: Usability Engineering Process

**5.1 Analysis**

**5.1.1 Intended Use**
- 의료기기의 의도된 사용
- 사용자 그룹 식별
- 사용 환경 정의
- Clinical indication 정의

**5.1.2 User Groups**
- Healthcare professionals (전문가)
- Lay users (일반인)
- Caregivers (간병인)
- Maintenance personnel (정비사)

**User Characteristics**:
- Age, education, training
- Physical limitations (vision, hearing, dexterity)
- Cognitive limitations
- Language proficiency
- Cultural differences

**5.1.3 Use Environments**
- Hospital setting
- Home setting
- Ambulatory setting
- Emergency setting
- Laboratory setting

**Environmental Factors**:
- Lighting
- Noise level
- Space constraints
- Temperature/humidity
- Sterility requirements

**5.1.4 User Interface Specification**
- UI 요구사항 정의
- UI 제약사항 식별
- UI 목표 설정

**5.2 Identification of Use Errors**

**5.2.1 Hazard-Related Use Errors**
- Use error → Hazardous situation → Harm
- ISO 14971 risk management process 연동
- Critical use errors 식별

**5.2.2 Use Error Identification Methods**
- Literature review
- Similar device analysis
- Use error databases
- Brainstorming
- Task analysis
- Walkthroughs

**Common Use Errors**:
- Misinterpretation of displays
- Confusion between similar controls
- Skipping important steps
- Incorrect assembly
- Wrong mode selection
- Parameter entry errors

**5.3 Design**

**5.3.1 UI Design Principles**
1. **Clarity**: Clear and unambiguous
2. **Consistency**: Consistent with user expectations
3. **Feedback**: Immediate and clear feedback
4. **Simplicity**: Minimize cognitive load
5. **Error Prevention**: Prevent use errors
6. **Error Recovery**: Easy error recovery

**5.3.2 UI Design Techniques**
- Task analysis
- Workflow design
- Information architecture
- Visual hierarchy
- Color coding
- Icon design
- Typography
- Layout design

**5.3.3 UI Design Outputs**
- UI specifications
- UI prototypes
- UI mockups
- UI storyboards
- UI style guide

**5.4 Validation**

**5.4.1 Usability Validation Plan**
- Validation objectives
- Validation methods
- Test participants
- Test scenarios
- Success criteria

**5.4.2 Usability Validation Methods**
- Formative evaluation (개발 중)
- Summative evaluation (개발 후)
- Expert review (Heuristic evaluation)
- User testing (실제 사용자)
- Simulation testing
- Clinical testing

**5.4.3 Usability Test Protocols**
1. **Recruitment**: Representative users
2. **Informed Consent**: Ethics compliance
3. **Training**: Appropriate training level
4. **Test Scenarios**: Realistic use cases
5. **Data Collection**: Think-aloud protocol
6. **Observation**: Direct observation
7. **Debriefing**: Post-test interview

**5.4.4 Usability Test Metrics**

**Objective Metrics**:
- Task completion rate
- Task completion time
- Error rate (per task)
- Error recovery rate
- Path efficiency
- Number of assists needed

**Subjective Metrics**:
- User satisfaction (SUS, QUIS)
- Perceived workload (NASA-TLX)
- Perceived safety
- Ease of use rating
- Preference ranking

**5.4.5 Acceptance Criteria**
- All critical tasks completed successfully
- No use errors leading to harm
- Error recovery rate > 90%
- User satisfaction > 80%
- No residual unacceptable risks

## Use Risk Management

### Integration with ISO 14971

**Use Error → Hazard Identification**
- Use error 식별 (IEC 62366)
- Hazard 식별 (ISO 14971)
- Risk estimation (ISO 14971)

**Use Error → Risk Control**
- UI design through risk control (IEC 62366)
- Risk control measures (ISO 14971)
- Residual risk evaluation (ISO 14971)

**Use Error → Validation**
- Usability validation (IEC 62366)
- Risk control effectiveness verification (ISO 14971)

## Usability Engineering File (UEF)

**Required Contents**:

1. **Usability Engineering Plan**
   - Scope, activities, responsibilities
   - User groups, use environments
   - UI specifications

2. **Use Error Analysis**
   - Identified use errors
   - Risk estimation

3. **UI Design**
   - UI specifications
   - Design rationale
   - Traceability to risks

4. **Usability Validation Report**
   - Validation plan
   - Test results
   - Acceptance criteria
   - Conclusion

5. **Traceability Matrix**
   - Use errors → UI design → Validation

## Common UI Design Mistakes

1. **Inconsistent Terminology**: Different terms for same concept
2. **Poor Color Contrast**: Hard to read displays
3. **Small Touch Targets**: Difficult to tap on touchscreens
4. **Confusing Icons**: Unclear icon meanings
5. **Hidden Functions**: Features not discoverable
6. **Overwhelming Displays**: Too much information
7. **No Feedback**: Unclear if action was accepted
8. **Hard to Recover**: No undo function
9. **Poor Error Messages**: Vague error descriptions
10. **Inconsistent Workflows**: Different flows for similar tasks

## UI Design Best Practices

### Physical Controls

- **Button Design**: Large, tactile, clearly labeled
- **Switch Design**: Clear on/off indication
- **Knob Design**: Directional markings
- **Connector Design**: Keyed, color-coded
- **Labeling**: High contrast, legible fonts

### Display Screens

- **Font Size**: Minimum 12pt for critical information
- **Color Coding**: Consistent color meanings
- **Layout**: Group related information
- **Animation**: Minimal, purposeful
- **Responsiveness**: < 1 second response time

### Touchscreens

- **Touch Targets**: Minimum 9mm × 9mm
- **Spacing**: Minimum 2mm between targets
- **Feedback**: Immediate visual/haptic feedback
- **Gesture Support**: Intuitive gestures
- **Error Prevention**: Confirmation for critical actions

### Alarms and Alerts

- **Priority Levels**: Clear priority indication
- **Audio Signals**: Distinct sounds per priority
- **Visual Indicators**: Flashing, color coding
- **Silence Options**: Time-limited silence
- **Clear Messages": Actionable alarm messages

### Labeling and Instructions

- **Concise**: Brief, clear text
- **Active Voice**: Direct instructions
- **Visual Aids**: Diagrams, icons
- **Warnings**: Prominent, specific
- **Language**: Local language support

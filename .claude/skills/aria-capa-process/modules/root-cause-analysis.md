# Root Cause Analysis (근본 원인 분석)

## 개요

Root Cause Analysis(RCA)는 불일치나 문제의 근본 원인을 체계적으로 식별하는 프로세스입니다.

## 5 Whys 기법

### 절차
1. 문제를 명확히 정의
2. "왜?" 질문을 5번 반복
3. 각 단계에서 답변 기록
4. 근본 원인 도출

### 예시
- Why 1: 센서 교정 실패 → 기준값 초과
- Why 2: Reference 만료 → 추적 시스템 없음
- Why 3: 프로세스 미정의 → SOP 부재
- Why 4: SOP 필요성 인식 부족
- Why 5: 교육 부족 → 근본 원인

## Fishbone Diagram

### 5M Categories
1. **Man**: 사람, 교육, 자격
2. **Machine**: 장비, 유지보수
3. **Material**: 자재, 원자재
4. **Method**: 절차, 프로세스
5. **Environment**: 환경, 조건

## FMEA-based RCA

### 절차
1. Failure Mode 식별
2. Effect 분석
3. Criticality 평가
4. Root Cause 도출
